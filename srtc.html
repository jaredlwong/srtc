Standalone WebRTC

What is WebRTC?

WebRTC is a method of communicating data over the internet, usually in your browser, between two clients without the need for a central server. In other words, it is a peer-to-peer communication link.

What is this page?

This page hosts the fundamental operations of WebRTC highlighting its serverless/standalone behavior.

The fundamental operations of WebRTC are video+audio streaming, and data communciation (such as instant messaging and file sharing).

Down below is a simple example showing how WebRTC can operate without a central server (kindof, still need to the ice servers). You can do text, audio and video communication.

Browsers

Chrome is the only browser I've gotten to work. Partly it is due to a hack I rely on for getting ice candidates (I wait for a null ice candidate, which is not in the spec, it's just what chrome happens to do).

Steps

  1. Contact a friend. Tell them to go to this webpage.
  2. Decide on the same set of communications.
  3. One person should press "Start"
  4. The person who pressed "Start" should send the text that will be generated in the first box to the other person. This could be through email.
  5. The other person should input the text from the person who pressed "Start" into the first box.
  6. The other person will get some generated text in the second box.
  7. The other person should send that text to the person who pressed "Start".
  8. The person who pressed "Start" should put the text from the other person's second box into his own second box.
  9. In the end, both people will have the same text in each box
  10. Now communicate!

Code

If you are interested you can find the code here and here. Note that communications are going through a single server that is in charge of routing. That would be the ice servers, in particular, stun:stun.l.google.com:19302.

But the ice servers just do the simple routing, setting up the NATs in between. They do not see any relevant information about the two parties involved beyond their IP addresses.

Notably, this is still a minor security vulnerability. Otherwise data is communicated directly between the parties with 128-bit AES.
